Node & V8 engine

JavaScript was created in 1995 and has since been known to be the dominant language for web development. For much of its initial days JavaScript was used mainly for client-side scripting inside <script> tags executing in web browsers. Due to this limitation, developers were often using different languages and frameworks between the front-end and backend aspects of a web application.

Node

Node.js came into existence in 2009 when the original developers of JavaScript extended it from something you could only run in the browser to something you could run on your machine as a standalone application.

Here's a formal definition for Node.js, per the Node.js website:

Node.js® is a JavaScript runtime built on Chrome's V8 JavaScript engine.

Node.js uses an event-driven, non-blocking I/O model that makes it lightweight and efficient.

Node allows developers to write JavaScript code that runs directly in a computer process itself instead of in a browser. Node can be used to write server-side applications with access to the operating system, file system, and everything else required to build fully-functional applications.

Features of Node.js

Following are some of the important features that make Node.js the first choice of developers.

Asynchronous and Event Driven

All APIs of Node.js library are asynchronous, that is, non-blocking. It essentially means a Node.js based server never waits for an API to return data.

Very Fast

Being built on Google Chrome's V8 JavaScript Engine, Node.js library is very fast in code execution.

Single Threaded but Highly Scalable

Node.js uses a single threaded model with event looping. Node.js uses a single threaded program and the same program can provide service to a much larger number of requests than traditional servers like Apache HTTP Server.

No Buffering

Node.js applications never buffer any data. These applications simply output the data in chunks.

License

Node.js is released under the MIT license.

V8 Engine

A JavaScript engine is a program or an interpreter which executes JavaScript code. A JavaScript engine can be implemented as a standard interpreter, or just-in-time complier that compiles JavaScript to bytecode in some form.

Here's a formal definition for V8, per the V8 Official website:

V8 is Google’s open source high-performance JavaScript and WebAssembly engine, written in C++. It is used in Chrome and in Node.js, among others.

V8 compiles and executes JavaScript source code, handles memory allocation for objects, and garbage collects objects it no longer needs. V8’s stop-the-world, generational, accurate garbage collector is one of the keys to V8’s performance.

V8 was first designed to increase the performance of JavaScript execution inside web browsers. In order to obtain speed, V8 translates JavaScript code into more efficient machine code instead of using an interpreter. It complies JavaScript code into machine code at execution by implementing a JIT (Just-In-Time) Compiler like most of the modern JavaScript engines.

Both JavaScript and Node.js run on the V8 JavaScript runtime engine. This engine takes your JavaScript code and converts it into a faster machine code. Machine code is low-level code which the computer can run without needing to first interpret it.

How Node.js uses V8 engine

Let's first understand how JavaScript is compiled and Run

JavaScript Program (source code) → Chrome V8 Engine (complied) → Machine Code (bytecode) → Runs on CPU

Different CPU architecture requires different machine code based on its architecture. So, the program needs to be converted into the machine code the machine can understand. Javascript program is compiled and the bytecode (machine code) is optimized and finally, the machine code runs on the CPU. All these operations are performed by the V8.

Let's see where the JavaScript resides based on the closeness to the computer hardware.

Javascript → C++ → Assembly Language → Machine Code

JavaScript is more high level than C++ and C++ is more than the Assembly language. Closer the language to the machine better is its performance and less is its compilation complexity.

Chrome V8 engine is really fast because it is written in C++. It also manages the memory allocation for the JavaScript programs. Allocating and deallocating memory to the objects as well as managing the threads stack memory is the responsibility of the V8 engine.

Now we know the role Chrome V8 engine plays in Node.js for making JavaScript reall fast and performant.
