Tutorial over API's

APIs

Application Programming Interfaces (APIs) are constructs made availabe in programming languages to allow developers to create complex functionality more easily. APIs provide easy syntax to use taking the complex code away from developers.

As a real-world example, think abou tthe electricity supply in your house. If you want to use an appliance in your house, you plug it into a plug socket and it works. You don't try to wire it directly into the power supply. A plug socket makes the work easier by taking away the complex wiring system. Same with APIs.

What can APIs do?

There are a huge number of APIs available in modern browsers that allow you to do a wide variety of things in your code. Most common categories of browser APIs are

APIs for manipulating documents

The most obvious example is the DOM (Document Object Model) API which allows you to manipulate HTML and CSS - creating, removing and changing HTML.

APIs that fetch data from the server

APIs that fetch data from the server are used to update small sections of a webpage on their own and are very commonly used. This seemingly small detail has had a huge impact on the performance and behaviour of sites.

Audio and Video APIs

Audio and Video APIs like HTMLMediaElement, the web Audio API allow you to do really interesting things with multimedia such as creating custom UI controls for playing audio and video, displaying text tracks like captions and subtitles.

Client-side storage APIs

Client-side storage APIs are becoming a lot more widespread in web browsers - the ability to store data on the client-side is very useful if you want to create an app that will save its state between page loads, and perhaps even work when the device is offline.

Web service APIs

There are a lot of different types of APIs, but we will discuss Web service APIs in general

A web service is a system or software that uses an address, i.e., URL on the World Wide Web, to provide access to its services.

The following are the most common types of web service APIs:

SOAP (Simple Object Access Protocol)

This is a protocol that uses XML as a format to transfer data. Its main function is to define the structure of the messages and methods of communication.

XML-RPC

This is a protocol that uses a specific XML format to transfer data compared to SOAP that uses a proprietary XML format. It is also older than SOAP. XML-RPC uses minimum bandwidth and is much simpler than SOAP.

JSON-RPC

This protocol is similar to XML-RPC but instead of using XML format to transfer data it uses JSON.

REST (Representational State Transfer)

REST is not a protocol like the other web services, instead, it is a set of architectural principles. Like all other architectural style, REST has it's own guiding principles which must be satisfied if an interface needs to be referred as RESTful. These principles are:

How to use an API

We will try to keep things simple, so to demonstrate examples I will be using OpenTable API - API interface to search OpenTable restaurant data.

CRUD and types of requests

CRUD is a programming concept denoting four basic actions Create, Read, Update and Delete that can be performed on a data source. In a REST API, these actions correspond to types of requests or request methods. These are four most commonly used requests:

  • POST: Create action. Adds new data to the server.

  • GET: Read action. Retrieves information. This is the most common type of request. Using this, we can get the data we are interested.

  • PUT: Update action. Changes existing information. Using this request, it would be possible modify or overwrite existing data.

  • DELETE: Delete action. Deletes existing information.

Endpoints

In order to work with REST APIs, it is important to understand the Endpoint concept. Endpoint is a specific address, by referring to which you get access to certain features/data. Commonly, the name of the endpoint corresponds to the functionality it provides.

Let's see an example of API request in JavaScript...


          // API request in JavaScript using a XMLHttpRequest
          let requestURL = 'http://opentable.herokuapp.com/api/cities';
          let request = new XMLHttpRequest();
          request.open('GET', requestURL);
          request.responseType = 'json';
          request.send();

          request.onload = function() {
            const responseObject = JSON.parse(this.responseText)
            console.log(responseObject) // prints the response from API which is an object that contains all cities in the API
          }

If you see in the above example, variable requestURL holds a link also known as an Endpoint. Using this endpoint we get cities data. Most of the times endpoint names are self-explanatory to the data they return.

          // API request in JavaScript using a XMLHttpRequest
          var requestURL = 'http://opentable.herokuapp.com/api/restaurants?zip=61761';
